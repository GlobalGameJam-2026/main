# 레벨 및 시각 문제 해결 가이드

## 🐛 발견된 문제들

1. **레벨이 너무 쉬움** - 가면 전환 없이도 통과 가능
2. **배경색이 안 바뀜** - 플랫폼만 색이 바뀌고 배경은 그대로
3. **Goal 도달 불가** - Goal이 플랫폼 위에 있어서 충돌 안 됨

---

## ✅ 해결 방법

### 1. 배경색 변경 문제 해결

**문제**: MaskManager의 배경색 설정이 잘못됨

**해결**:

1. **Hierarchy에서 MaskManager 선택**

2. **Inspector에서 설정 확인**:
   - **Current Mask**: White (기본값)
   - **Black Mask Background Color**: 
     - R: 0, G: 0, B: 0, A: 255 (완전한 검은색)
   - **White Mask Background Color**: 
     - R: 255, G: 255, B: 255, A: 255 (완전한 흰색)
   - **Main Camera**: Main Camera 드래그해서 할당

3. **테스트**:
   - Play 누르고 Shift 키로 가면 전환
   - 배경이 흰색 ↔ 검은색으로 바뀌어야 함

---

### 2. 레벨 데이터 재생성 (간격 추가)

**문제**: 플랫폼 간격이 좁아서 가면 전환 없이도 점프 가능

**해결**:

1. **LevelCreator 선택**

2. **우클릭 메뉴**:
   - **Create Stage 1 Data** 클릭 → Yes
   - **Create Stage 2 Data** 클릭 → Yes

3. **Console 확인**:
   - "MANDATORY mask switching with gaps" 메시지 확인

**변경 사항**:
- 플랫폼 간격을 3~4 유닛으로 확대
- 잘못된 가면으로는 **절대 점프 불가능**
- 각 점프마다 가면 전환 필수

---

### 3. Goal 위치 수정

**문제**: Goal이 플랫폼과 같은 높이에 있어서 충돌 감지 안 됨

**해결**: 이미 코드에서 자동 수정됨!
- Goal이 플랫폼보다 0.75 유닛 위에 생성됨
- 레벨 데이터 재생성하면 자동 적용

---

## 🎮 테스트 체크리스트

### 배경색 테스트
1. Play 버튼
2. Shift 키로 가면 전환
3. **확인**: 
   - ✓ 흰 가면 → 흰 배경
   - ✓ 검은 가면 → 검은 배경
   - ✓ 플랫폼 색상도 반대로 변경

### 레벨 난이도 테스트
1. Play 버튼
2. **흰 가면**으로 시작
3. **확인**:
   - ✓ 첫 번째 플랫폼 보임 (WhiteOnly)
   - ✓ 두 번째 플랫폼 안 보임 (BlackOnly)
   - ✓ Shift 눌러서 검은 가면으로 전환
   - ✓ 이제 두 번째 플랫폼 보임
   - ✓ 각 점프마다 가면 전환 필수

### Goal 도달 테스트
1. 마지막 플랫폼까지 이동
2. **확인**:
   - ✓ Goal이 플랫폼 위에 떠 있음
   - ✓ Goal에 닿으면 "🎉 Goal Reached!" 메시지
   - ✓ 2초 후 Stage 2로 이동

## 🎨 새로운 레벨 디자인

### Stage 1
- **플랫폼 개수**: 8개
- **간격**: 2.5~3 유닛 (점프 거리 한계)
- **특징**: 
  - 각 플랫폼마다 가면 전환 필수
  - 잘못된 가면으로는 플랫폼 안 보임
  - 보이지 않는 플랫폼으로 점프 = 낙사

### Stage 2
- **플랫폼 개수**: 9개
- **간격**: 3~4 유닛 (매우 큰 간격)
- **특징**:
  - 플랫폼 크기 1 유닛 (매우 작음)
  - 정밀한 점프 필요
  - 빠른 가면 전환 필수

## 💡 추가 팁

### 플랫폼이 잘 안 보인다면
- MaskObject 컴포넌트 확인
- Black Mask Color: 흰색 (255, 255, 255)
- White Mask Color: 검은색 (0, 0, 0)

### 점프가 안 닿는다면
- TaroDev 컨트롤러의 Jump Height 증가
- Player → Inspector → Scriptable Stats → Jump Power 조절

### 난이도 조절
- LevelDataCreator.cs 파일 열기
- 플랫폼 간격 조절 (x 좌표 차이)
- 간격 작게 = 쉬움, 간격 크게 = 어려움
